## 项目计划：日本麻将理论最大得点搜索算法设计与分析

### 一、课题目标

开发一个确定性搜索工具，给定当前牌局状态（手牌、牌河、宝牌指示牌、副露、巡目），在合理约束下计算出**理论上能达成的最大番数**（含累计役满和复合役满），并输出达成该牌型所需的关键牌序及鸣牌建议。

### 二、输入与输出

- **输入**：
  - 手牌（14张或13张+摸牌前）
  - 副露（碰/杠/吃的组）
  - 各家牌河（已打出的牌，可简化只记录数量）
  - 宝牌指示牌（表宝牌、里宝牌可选）
  - 当前巡目（可选）
- **输出**：
  - 理论最大番数（如“双倍役满”或“累计役满 17 番”）
  - 达成该番数需要的摸牌序列（最多5步）
  - 推荐的鸣牌动作（碰/杠/吃）
  - 剪枝前后的搜索节点数对比（供复杂度分析）

### 三、核心算法框架

#### 1. 状态表示

- 牌编码：`0~33` 分别代表 `1m~9m, 1p~9p, 1s~9s, 东南西北白发中`
- 手牌：长度为34的整数数组，表示每种牌的数量（0~4）
- 副露：结构体数组，记录每副露的类型（碰/杠/吃）及牌
- 剩余牌：`rest[34]`，初始值为4，减去所有已见牌（手牌+副露+牌河+宝牌指示牌）

#### 2. 搜索策略：深度优先搜索 + 分支定界

- 搜索深度限制：`max_draw` = min( 剩余巡目, 5 )，默认5
- 每层递归：
  - 尝试摸入每种仍有剩余（`rest[牌]>0`）的牌
  - 更新手牌后，检查是否可和牌（调用番数计算器）
  - 若可和牌，记录番数并更新全局最优
  - 若不可和牌，尝试所有合法的鸣牌动作（碰、杠、吃，可选），生成新状态
  - 剪枝：计算乐观估计上限 `current_score + optimistic_bonus(state)`，若 ≤ 当前最优则剪枝
- 返回最优番数及对应路径

#### 3. 番数计算模块

- 役种判定：实现主要役满（国士、九莲、四暗刻、大三元、小三元、清一色、混一色）及部分高番役种（如一气通贯、三色同顺）
- 复合役满：累加多个役满，超过13番记为累计役满
- 支持门清与非门清判定
- 拆牌DP：对手牌划分顺子/刻子，用于普通番种（可选，为了简化可先做役满特化）

#### 4. 乐观估计函数 `optimistic_bonus(state)`

- 方法：针对每个役满牌型，计算当前还缺几张关键牌；若缺牌数 ≤ 剩余摸牌次数，则累加该役满的番数
- 示例：国士无双缺3种幺九牌，剩余摸牌次数=5，则可乐观加上役满；四暗刻当前有3个对子且剩余摸牌次数≥2，则可乐观加上役满
- 注意：防止重复累加互斥役满（如国士与九莲），需查互斥表

### 四、具体约束设计（硬约束+软剪枝）

| 约束                            | 类型     | 实现方式                                                 |
| ------------------------------- | -------- | -------------------------------------------------------- |
| 每张牌最多剩余4张               | 硬       | `rest[34]` 初始4，遇牌减1                                |
| 最大摸牌次数 = min(剩余巡目, 5) | 硬       | 搜索深度限制                                             |
| 副露后排除非门清役满            | 逻辑剪枝 | 若副露非空，目标集中移除国士、九莲、四暗刻               |
| 对子数上限（四暗刻）            | 启发剪枝 | 当前对子数 + floor(剩余摸牌次数/2) < 4 → 剪掉四暗刻分支 |
| 役满互斥表                      | 逻辑剪枝 | 枚举组合时检查，如国士与九莲不可共存                     |
| 花色优选（清一色）              | 优化     | 只对手牌数量最多的花色进行清一色搜索                     |

### 五、实现步骤（分阶段）

#### Phase 1：基础框架（1-2天）

- 实现牌编码、剩余牌统计、手牌增删操作
- 实现简单的深度优先搜索（无剪枝，最大深度3），仅支持国士无双判定
- 验证：人工构造几个已知听国士的牌例，检查输出是否正确

#### Phase 2：添加役满牌型（2-3天）

- 实现九莲宝灯、四暗刻、大三元、清一色判定
- 添加番数累加逻辑
- 加入最大摸牌次数约束（固定深度5）

#### Phase 3：剪枝与优化（2天）

- 实现乐观估计函数
- 加入对子数上限、副露后过滤、役满互斥等约束
- 测试对比剪枝效果（记录节点数）

#### Phase 4：鸣牌决策（可选，1-2天）

- 实现碰/杠/吃的动作生成（注意吃只能顺子，且需指定从哪家）
- 在搜索中允许选择鸣牌，更新手牌和副露状态
- 注意：鸣牌会消耗牌河中的牌，需同时更新 `rest` 和手牌数

#### Phase 5：界面与报告（1-2天）

- 命令行交互：手动输入手牌、牌河等
- 输出最优番数、关键牌序列、鸣牌建议
- 收集运行数据（搜索节点数、耗时），用于论文复杂度分析

### 六、复杂度分析要点（论文内容）

- **状态空间上界**：每个深度最多尝试34种摸牌 × 若干鸣牌动作，深度5，上界约 (34×3)^5 ≈ 10^9，但实际剪枝后通常 < 10^5 节点
- **时间**：单次番数计算 O(1)（役满模板匹配），整体在 0.1~0.5 秒内
- **空间**：递归栈深度≤5，状态拷贝开销可接受
- **对比实验**：展示有无剪枝的节点数差异（如国士无双例：无剪枝10万节点，有剪枝2000节点）
